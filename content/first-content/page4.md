Построение интерфейса
---------------------

### Алгоритмическое формирование интерфейса

В АИС реализована возможность настройки экранных форм путем указания
состава и правил отображения элементов интерфейса на странице:

1.  Тип элемента, его вид и состав (если элемент может быть разных видов
    и содержания).

2.  Свойства элемента (набор свойств зависит от типа). Например, для
    элементов ввода данных: название реквизита, тип данных, признак
    обязательности к заполнению, тексты подсказок и сообщений об
    ошибках.

3.  Расположение: с новой строки или продолжает текущую.

4.  Минимальный и максимальный размер (количество колонок в сетке) в
    рамках ограничений, предустановленных типом данных.

### Оптимальная плотность информации

В интерфейсе АИС соблюдается баланс между плотностью информации и
удобством использования:

-   Максимум площади экрана отводится под содержание. Для этого блоки
    навигации (боковое меню, вкладки карточек) и настроек (панель
    фильтрации) могут быть свернуты пользователем после использования;

-   При размещении полей фильтрации и ввода данных, а также более
    крупных блоков рекомендуется использовать вертикальные отступы в
    диапазоне от 16 px до 32 px;

-   Рекомендуется использовать оптимальную ширину полей ввода
    информации, которая будет подсказывать пользователям формат
    ожидаемых в них данных (например, поля для ввода дат и короткого
    текста не должны быть слишком длинными);

-   При выводе результатов фильтрации рекомендуется показывать
    примененные фильтры в одну линию, а если значения не уместились в
    одну линию, то скрывать их в выпадающий список «Еще N».

### Автоматическая адаптация интерфейса под условия использования АИС ФССП России

Интерфейс АИС адаптируется под изменения технических характеристик АРМ и
качества связи:

-   При изменении размеров окна изменяется компоновка содержания (см.
    раздел 5.3 Модульная сетка) и представление кнопок (при уменьшении
    ширины окна: кнопки с содержанием «пиктограмма и текст» заменяются
    на пиктограммы, далее на месте нескольких кнопок показывается одна
    кнопка с пиктограммой «многоточие», по нажатию на которую
    открывается выпадающий список с действиями, которые выполняли
    скрытые кнопки);

-   При ожидании ответа от сервера на месте отображения данных
    выводится:

    -   Идентификатор загрузки (элемент spin);

    -   Скелет ожидаемых данных (обобщенная визуализация структуры
        данных -- таблица, поля ввода данных, карточки и т.д);

    -   Текстовое сообщение об ожидании загрузки данных.

### Персонализация интерфейса

Пользовательский интерфейс АИС максимально адаптируется под потребности
и предпочтения пользователей:

1.  Пользователю в интерфейсе АИС доступны только те элементы и функции,
    которые определены его ролью в АИС (в соответствии с правами
    доступа).

2.  Для ролей пользователей в АИС предварительно созданы настройки
    бокового меню (определен состав и порядок следования пунктов меню, в
    которых осуществляется основная работа пользователями этой роли), а
    также стартовая страница, которая открывается при входе пользователя
    в АИС.

3.  Пользователь может настраивать интерфейс в соответствии со своими
    обязанностями и предпочтениями:

    a.  Состав и порядок следования пунктов в боковом меню;

    b.  Стартовая страница, которая открывается при входе пользователя в
        АИС;

    c.  Состав и порядок следования виджетов на рабочем столе личного
        кабинета;

    d.  Настройки фильтров для списков;

    e.  Настройки списков -- выбор представления (таблица или карточки),
        состав данных в представлении (состав и порядок столбцов в
        таблице, состав карточек), состав области представления (состав
        и порядок секций и вкладок).

4.  АИС автоматически сохраняет пользовательские настройки, даже если
    пользователь не сохранил их в явном виде как шаблон. АИС отображает
    экраны в соответствии с теми настройками, которые пользователь
    применил при последнем просмотре этих экранов.

5.  При изменении настроенного пользователем элемента:

    a.  Если была добавлена характеристика, то эта характеристика
        отображается для пользователя;

    b.  Если был убран элемент или характеристика, то этот элемент или
        характеристика перестают отображаться для пользователя, и в
        уведомлениях появляется сообщение об изменении.

### Минимизация обращений к серверу

В целях снижения нагрузки на сервер и каналы связи веб-интерфейс АИС
должен обращаться к серверу при следующих действиях:

1.  При входе пользователя в АИС.

2.  При переходе пользователя к новому пункту меню (горизонтального или
    бокового).

3.  При работе пользователя со списками:

    a.  Открытие списка;

    b.  Переход по страницам списка;

    c.  Применение фильтров.

4.  При работе с карточками:

    a.  Открытие пользователем карточки;

    b.  Обновление карточки по запросу пользователя;

    c.  Изменение статуса карточки пользователем;

    d.  Автоматическое сохранение изменений АИС.

Навигация по АИС ФССП России
----------------------------

### Элементы интерфейса для навигации

Для глобальной навигации по АИС используются элементы:

1.  Горизонтальное меню -- для вызова бокового вертикального меню,
    перехода к профилю пользователя и настройкам, уведомлениям,
    обращению в техническую поддержку.

> Подробно горизонтальное меню описано в разделе 6.2.1.

2.  Боковое меню -- для перехода по функциональным модулям и разделам
    АИС.

> Подробно боковое меню описано в разделе 6.2.2.

3.  Хлебные крошки -- для перехода на предыдущую страницу (просмотренную
    пользователем до перехода на текущую), на другие страницы выбранного
    раздела, на страницы других разделов (навигация без вызова бокового
    меню).

### Использование клавиатуры для навигации

Для навигации по АИС поддерживается клавиатура:

-   Переход по элементам списков в виде таблиц (по строкам) -- путем
    нажатия стрелок вверх и вниз;

-   Переход по ячейкам одной строки в таблицах -- путем нажатия стрелок
    влево и вправо;

-   Переход по элементам ввода (переключение фокуса от поля к полю) -- с
    помощью клавиши Tab, минуя подсказки и вспомогательные элементы.

Подробно использование клавиатуры для навигации и управления
представлено в описании элементов интерфейса.

### Навигация с помощью элементов браузера

Для навигации по экранам АИС не рекомендуется использовать кнопки
браузера «Вперед» и «Назад», так как поведение этих кнопок может
зависеть от конкретного браузера и его версии.

При необходимости одновременной работы на нескольких экранах
рекомендуется использовать вкладки браузера. При этом в АИС
обеспечивается:

-   Обновление данных на вкладках при изменении этих же данных на других
    вкладках, кроме настроек фильтрации (один и тот же список может быть
    открыт в нескольких вкладах с разными параметрами фильтрации
    данных);

-   Уникальный адрес для каждой вкладки карточки (для возможности
    быстрого перехода и сохранения последней открытой вкладки карточки).

### Использование модальных окон

В АИС рекомендуется использовать модальные окна:

-   Для действий, которые не могут быть выполнены в фоне без блокировки
    работы родительской страницы;

-   Для выбора значений из справочников;

-   Для просмотра вложений на полный экран;

-   Рекомендуемое максимальное количество уровней вложенности модальных
    окон -- 2.

Использование средств ввода информации
--------------------------------------

### Мышь

В АИС поддерживается использование мыши для:

-   Навигации по АИС (выбор пунктов меню, нажатие ссылок, пролистывание
    содержания (скролл), переход по страницам);

-   Управления (нажатие кнопок, установка фокуса на элементах ввода
    данных и управления).

### Клавиатура

> В АИС используется клавиатура для навигации по принципу Mousless,

  Alt + X          Выйти

### Сканер штрих-кодов

> []{#_Toc42160344 .anchor}АИС должна поддерживать использование сканера
> штрих-кодов для:

-   Поиск документа, сканер штрих кодов сканирует штих код документа и
    > находит электронную версию документа.

-   Сравнение актуальности печатной версии документа и электронной.

### Тачпад (устройство ввода на мобильных устройствах)

АИС должна поддерживать использование тачпада на мобильных устройствах
для:

-   Ввода данных (заполнения фильтров, полей ввода),

-   Навигации по АИС (выбор пунктов меню, нажатие ссылок, пролистывание
    содержания (скролл), переход по страницам);

-   Управления (нажатие кнопок).

Автоматизация и ускорение действий пользователя
-----------------------------------------------

В АИС должны быть реализованы способы автоматизации и ускорения действий
пользователя:

-   Определение и демонстрация в интерфейсе ожидаемых от пользователя
    действий;

-   Выстраивание элементов интерфейса в соответствии с приоритетами
    ожидаемых действий, частотой использования функций, ролью в АИС и
    предпочтениями пользователей;

-   Сокрытие элементов, действия с которыми не требуются (вкладки
    карточек, поля для ввода данных), с возможностью их вызова
    пользователем при необходимости;

-   Автоматическое заполнение полей с возможностью редактирования
    значений пользователем.
